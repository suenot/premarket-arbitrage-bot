# Логика арбитража

## Принцип

Prediction market — бинарный рынок: событие либо случится (YES), либо нет (NO). При resolution одна сторона получает $1, другая — $0.

**Арбитраж** возникает когда на двух разных платформах цены на одно событие расходятся настолько, что:

```
YES_price(Platform_A) + NO_price(Platform_B) < $1.00
```

Покупая YES на платформе A и NO на платформе B, вы гарантированно получаете $1 при любом исходе, потратив меньше $1.

## Пример

| | Polymarket | Kalshi |
|---|-----------|--------|
| YES | $0.65 | **$0.50** ← дешевле |
| NO | **$0.38** ← дешевле | $0.55 |

**Стратегия**: Buy YES @ Kalshi ($0.50) + Buy NO @ Polymarket ($0.38)

```
Total cost = $0.50 + $0.38 = $0.88
Guaranteed return = $1.00
Profit = $0.12 (13.6%)
```

### Если событие СЛУЧИЛОСЬ (YES wins):
- YES @ Kalshi → получаем $1.00
- NO @ Polymarket → получаем $0.00
- **Итого: $1.00 - $0.88 = +$0.12 profit**

### Если событие НЕ СЛУЧИЛОСЬ (NO wins):
- YES @ Kalshi → получаем $0.00
- NO @ Polymarket → получаем $1.00
- **Итого: $1.00 - $0.88 = +$0.12 profit**

## Метрики

### profit / profit_pct
```
profit = 1.0 - total_cost
profit_pct = profit / total_cost × 100
```

### APR (Annual Percentage Rate)
```
apr = profit_pct × (365 / days_until)
```

Чем меньше дней до экспирации — тем выше APR при том же profit.

### depth (ликвидность)
`max_usd` — максимальная сумма, которую можно вложить по текущим ценам. Реальный slippage может быть больше при крупных ордерах.

## Риски

| Риск | Описание | Митигация |
|------|----------|-----------|
| **Execution risk** | Пока размещаете ордер на одной платформе, цена на другой может измениться | Быстрое исполнение, лимитные ордера |
| **Liquidity risk** | Недостаточная ликвидность для полного исполнения | Фильтр `MIN_DEPTH_USD` |
| **Platform risk** | Платформа может не выплатить | Диверсификация, проверенные платформы |
| **Fee risk** | Комиссии платформ уменьшают profit | Учёт `platform_fee` в расчётах |
| **Resolution risk** | Спорное resolution события | Чтение описания рынка |
| **Time risk** | Капитал заблокирован до resolution | Учёт `days_until`, APR vs absolute profit |

## Фильтры бота

Бот применяет 3 фильтра к парам из Premarket API:

1. `profit_pct >= MIN_PROFIT_PCT` — минимальная прибыль
2. `apr >= MIN_APR` — минимальная годовая доходность
3. `depth.max_usd >= MIN_DEPTH_USD` — минимальная ликвидность

Результат сортируется по APR по убыванию.
